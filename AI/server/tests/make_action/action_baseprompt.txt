**TASK**: For each agent, determine **ONE NEXT ACTION** based on their current state.

### üì• Input Information

* **name**: The agent's name
* **state**: The agent's condition, including:

  * `hunger`: 0‚Äì3 = not hungry, 4‚Äì6 = normal, 7‚Äì10 = very hungry
  * `sleepiness`: 0‚Äì3 = not sleepy, 4‚Äì6 = normal, 7‚Äì10 = very sleepy
  * `loneliness`: 0‚Äì3 = not lonely, 4‚Äì6 = normal, 7‚Äì10 = very lonely
* **current\_location**: The agent's current position
* **visible\_interactables**: Locations and objects currently visible to the agent

  * `location`: A visible place
  * `interactable`: Objects or characters visible within that location

### ‚úÖ Available Actions

* `move`: Travel to a new location
* `interact`: Use or manipulate an object
* `eat`: Consume food
* `talk`: Engage in conversation with another agent
* `wait`: Remain inactive for a short time
* `think`: Reflect or engage in internal thought
* `idle`: Do nothing
* `sleep`: Rest to recover energy

### ‚ö†Ô∏è Important Rules

* The map follows a **Region ‚Üí Location ‚Üí Object** hierarchy.
* The `location` and `target` in the output must come from the `visible_interactables`.
* If hunger ‚â• 7 and any visible interactable is food or food-related, prioritize action "eat" on that object.
* If no edible object is visible, and there is a known nearby location that contains food, choose action "move" toward that location.
